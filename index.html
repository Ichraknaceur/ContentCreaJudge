    ContentCreaJudge¶

    Project Snapshot

    ContentCreaJudge is a rule-based editorial evaluation engine with a FastAPI backend and a Streamlit demo UI. The current foundation is designed to let the team implement mini-judges independently while keeping one stable API contract.

    Why This Project Exists¶

    AI-assisted engine for evaluating and improving editorial content quality and compliance.

    Current V1 foundation already includes:

    • FastAPI backend bootstrap
    • GET /
    • GET /health
    • POST /api/v1/evaluations
    • Streamlit UI client aligned with the ContentCrea visual theme

    At A Glance¶

    • Backend --- FastAPI service exposing health, discovery, evaluations, and judge routes.

    • UI --- Streamlit workspace for demos, manual testing, and isolated judge checks.

    • Rules --- YAML-driven rule definitions kept separate from judge execution logic.

    • Team Docs --- Architecture, cadrage, API contract, and error-handling doctrine in one place.

    Setup and installation¶

    Prerequisites¶

    • Python 3.14 or higher
    • uv for dependency management

    Installation¶

    make init

    Usage¶

    make run

    The API is exposed locally at:

    • http://127.0.0.1:8000/
    • http://127.0.0.1:8000/health
    • http://127.0.0.1:8000/api/v1/evaluations
    make run-ui

    The UI is exposed locally at:

    • http://127.0.0.1:8501

    Current UI Scope¶

    The Streamlit UI currently provides:

    • backend connection status
    • API overview
    • evaluation payload form
    • response console
    • delivery view for client demos

    Recommended local workflow

    Start the backend first with make run, then start the UI with make run-ui. The judge playground depends on the API being available.

    Architecture¶

    The V1 service architecture follows a modular evaluation pipeline:

    Request -> Validation -> Rule Resolution -> Preprocessing -> Judges
-> Aggregation -> Response

    The current implementation also includes a presentation layer for demos:

    Streamlit UI -> FastAPI API -> Evaluation endpoint scaffold

    For the full design, module boundaries, and internal guidelines, see the architecture and developer guide pages linked above.
